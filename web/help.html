Introduction

Welcome to the Mail DataBase although it has expanded slightly now. Here is a list of all the things it does at the moment

Submit email from PCAP, Text stream, IMAP or POP feeds.

  • The email parser will read the headers extracting the main fields
  • Any attatchments are extracted, Hashed stored and scanned with Yara, optionally you can include options to scan with Clam AV and submit to Virus Total
  • All the information extracted above is stored in to a SQLite Database as a Task.
  • All files are stored for further analysis
  • Options to submit file attatchments to Sandboxs, Cuckoo or FireEyes, MAS

    Http PCAP files

  • Extract all the streams
  • Parse the HTTP Headers
  • Extract all the HTTP Objects and files
  • Replicate the Server Path structure
  • Write the Header data to the SQLite Database as a Task

    Management System

  • Micro management system allows you to track Tasks in the Database
  • Add Comments to Tasks
  • Set tasks as Reviewed, Events or Unchecked
  • Generate, Weekly, Montly, Yearly reports on all Submitted tasks
  • Trend analysis on all artefacts in the database

    • Configuration

    The config file is used to store user defined variables, allowing you to change file paths, and enable / disable options

    Most of the configuration settings use a 1 or a 0 to Enable or Disable options.

    Here are the items that can be configured

  • transferDir This is the temporary directory where uplaoded files are stored before processing
  • reportRoot, This is the directory that will store any files associated with a submitted task
  • DBFile, This is the location of the SQLITE database file. This config setting will allow for otehr SQL varients in the future
  • ProxyPort and SSLPort used for parsing the HTTP Traffic, if your not using a proxy use '0080' and '443'
  • ssdeep, Enable or Disable SSDEEP Hashing
  • clamscan and dblocation will set the Definition location and enable scanning of all attathments and http objects
  • enableYara speaks for itself. emailSig and fileSig set the file location for your Yara Rules
  • falsopos allows for a Yara Ruile File that contains known false postitives for quick review
  • Virus Total Enabel and API
  • Sandox needs updating to allwo for a more modular approach :)

    • Submitting Tasks

    Emails

    Email Tasks can be submitted in one of the following formats

  • PCAP File containing a single or multiple SMTP streams
  • Text Stream containg RFC 882 or equivelent stream
  • POP3 or IMAP Protocols, IMAP supports Defining a Folder, POP just process the Inbox.

    HTTP Traffic

    Before submitting HTTP PCAP files it is important to ensure the Proxy Port settings are configured correctly in the Config File

    The submitted file MUST be name http-xxxx.pcap, where xxxx can be any string.

    There is currently NO progress bar to indicate the processing of HTTP traffic. This process can be time consuming for Large PCAP files.

    Other Tasking

    When submittign other Taskings. i.e. files that can not be parsed by the DB, First compress the files in to a single Zip, RAR, or tar.gz file.

    The submitted file MUST be nameed Tasking-xxxx, where xxxx can be any string.

    Submit the file using the upload form and add a comment appropriate to the Task. e.g Tasking-Department Name

    For all submitted tasks there is an optional field Called Comments. This can be used to add a Tag that is used as par tof the reporting process. This tag uses the keyword 'Tasking' to produce reports.

    Any other keyword can be used e.g. a Snort Rule Name that will be searchable using the search Options
